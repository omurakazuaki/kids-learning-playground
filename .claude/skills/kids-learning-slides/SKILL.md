# 子ども向け学習スライド生成スキル

このスキルは、子ども向け学習コンテンツを Slidev 形式の Markdown として生成し、
`docs/<年齢>/<カテゴリ>/<ファイル名>.md` に保存する。

## 入力の確認

ユーザーから以下を確認する（不明な場合は質問する）：
- **対象年齢**（例: 5歳、6歳、8歳など）
- **カテゴリ**（算数、国語、英語、推論など）
- **問題の内容・テーマ**（ユーザーが提示するか、カテゴリに応じて自動生成）

## 保存先

```
docs/<年齢>/<カテゴリ>/<テーマを表す短い英語スネークケース名>.md
```

例: `docs/6/reasoning/broken_clock.md`、`docs/8/math/multiplication.md`

ファイルが存在する場合は確認してから上書きする。

## スライド構成（4セクション）

各ファイルは次の4つのセクションをスライドとして含む。セクション間は `---` で区切る。

### 1. 事前学習シート（Pre-learning）
- このセクションの目的: 問題を解くために必要な前提知識や考え方を、視覚的・わかりやすく提示する
- 「まず知っておこう！」のような導入
- 図・例・ステップで考え方を示す
- 問題そのものは出さない

### 2. 問題シート（Problem）
- 問題文を提示する（読みやすいフォント、余白、絵文字で視覚的に）
- 解答欄は「こたえ：＿＿＿＿」のように空白スペースを設ける
- 解答は伏せる（このシートには書かない）

### 3. ヒントシート（Hint）
- 問題を直接解かずに「考え方のヒント」を提示する
- 段階的に気づきを促す（ヒント1、ヒント2…）
- 「どうすればわかるかな？」という問いかけ形式が効果的

### 4. 回答・解説シート（Answer & Explanation）
- 正解を明示する
- なぜその答えになるかを丁寧に解説する
- 子どもが「なるほど！」と感じられる説明を心がける

## 年齢別ガイドライン

| 年齢 | 口調・表現 | 文字 | 難易度 |
|------|-----------|------|--------|
| 5〜6歳 | ひらがな中心、「〜だよ」「〜だね」、単語は分かち書き | 漢字なし or 全ルビ | とても簡単。日常の具体的なもの |
| 7〜8歳 | やさしい漢字、「〜です」「〜しよう」 | 小学1〜2年漢字にルビ | 簡単〜普通。絵や図で補助 |
| 9〜10歳 | 普通の文体、「〜しましょう」 | 小学3〜4年漢字にルビ | 普通〜やや難しい |
| 11〜12歳 | 標準語、説明的な文章も可 | 小学5〜6年漢字にルビ | やや難しい〜難しい |

**分かち書き（5〜6歳向け）**: 単語間にスペースを入れる。例: `とけい が こわれた よ！`

## ルビのルール

すべての漢字にルビを振ること。HTML の `<ruby>` タグを使う：

```html
<ruby>算数<rt>さんすう</rt></ruby>
<ruby>問題<rt>もんだい</rt></ruby>
```

5〜6歳向けはできる限り漢字を避け、ひらがなで書く。

## Slidev フォーマット

各スライドファイルのフロントマター（先頭）:

```yaml
---
theme: default
title: <タイトル>
transition: slide-left
mdc: true
---
```

スライドの分割は `---` で行う。各スライドは1つのトピックに絞り、テキストを詰め込みすぎない。

スタイルのヒント（Slidev では `<style>` タグを各スライド内に書ける）:
- **フォントサイズ**: Slidev のスライドは縦が限られているので、はみ出しに注意する
  - 本文: `1.0rem`〜`1.1rem`（強調ブロックでも `1.2rem` 止まり）
  - `1.5rem` 以上は見出し（`#`）のみに留める
- **line-height**: `1.6`〜`2.0` を目安にする（`2.4` 以上は詰まりすぎの原因）
- **padding / margin**: `0.6em`〜`1em` 程度に抑える
- 絵文字を活用して視覚的に楽しくする
- 色はカテゴリや気分に合わせて変える
- 1スライドに要素を詰め込みすぎないこと。収まらない場合はスライドを分割する

## インタラクティブ要素（任意）

問題や答え合わせに Vue コンポーネントを使ってよい（Slidev は `<script setup>` をサポート）。
例: ボタンを押すと答えが表示される、チェックボックスで選択肢を選ぶ。
ただしシンプルさを優先し、複雑になりすぎないようにする。

## 生成の流れ

1. ユーザーの入力から年齢・カテゴリ・テーマを確認
2. 問題内容を設計（または確認）する
3. 4セクション（事前学習・問題・ヒント・解説）のスライドを作成
4. `docs/<年齢>/<カテゴリ>/<ファイル名>.md` に保存
5. **生成・編集後に必ず `pnpm lint` を実行**してエラーがないか確認する（下記参照）
6. ファイルパスをユーザーに報告し、`pnpm dev` で確認できる旨を伝える

## 事前チェック（lint）

スライドを生成・編集したら、コミット前・報告前に必ず以下を実行する：

```bash
pnpm lint
```

`scripts/lint-slides.mjs` が `docs/**/*.md` を走査し、以下を検出する：

- **タグ閉じ漏れ**: `<` から次の `>` までに別の `<` が現れるケース
  - 例: `<ruby>力<rt>ちから</rt</ruby>` のような `</rt` の `>` 抜けを検出
- **`<ruby>` ブロックの構造不正**: `<rt>...</rt>` を含まない `<ruby>` ブロック
- **タグの開閉個数不一致**: `<ruby>`/`</ruby>`、`<rt>`/`</rt>`、`<div>`/`</div>`、`<span>`/`</span>`
- **frontmatter の欠落**: 先頭の `---` ブロックや `theme`/`title`/`mdc` キー

エラーが出たら**必ず修正してから報告**する。lint は CI（`.github/workflows/deploy.yml`）でも
ビルド前に実行されるため、通っていないと GitHub Pages へのデプロイが失敗する。

`slidev build` での確認は最終手段（時間がかかる）。lint で 99% のケースを事前に弾けるので、
通常は lint だけで十分。

## 出力例（6歳向け推論）

```markdown
---
theme: default
title: こわれた とけい
transition: slide-left
mdc: true
---

# ⏰ まず <ruby>知<rt>し</rt></ruby> っておこう！

とけい の はり は、とまった しゅんかん の じかん を さしたまま になるよ。

---

## ❓ もんだい

...（問題文）...

こたえ：＿＿＿＿ さん

---

## 💡 ヒント

ヒント１: とけい の はり は **「３じ」** で とまって いるね。...

---

## ✅ こたえ と かいせつ

こたえ：**ぶたさん** 🐷

なぜなら...
```

---

問題の質（子どもが楽しめる、達成感がある、教育的価値がある）を最優先に考えて生成すること。
スライドはただ情報を並べるのではなく、子どもが「やってみたい！」「わかった！」と感じられる体験を作ることを意識する。
